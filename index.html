바이브코딩 완벽 매뉴얼

Claude · GitHub · Supabase · Vercel · VS Code

현대 개발자를 위한 5가지 핵심 도구의 완벽한 사용 가이드. 설치부터 고급 활용까지 한 곳에서 확인하세요.

Claude

Anthropic의 AI 어시스턴트. 코딩, 글쓰기, 분석 등 다양한 작업을 도와주는 강력한 AI 도구입니다.

GitHub

세계 최대의 코드 호스팅 플랫폼. 버전 관리, 협업, CI/CD를 위한 필수 개발 도구입니다.

Supabase

오픈소스 Firebase 대안. PostgreSQL 기반의 백엔드 서비스로 인증, DB, 스토리지를 제공합니다.

Vercel

프론트엔드 배포의 표준. GitHub 연동 자동 배포, 전 세계 엣지 네트워크를 제공합니다.

VS Code

Microsoft의 강력한 코드 에디터. 방대한 확장 프로그램과 AI 통합으로 생산성을 극대화합니다.

Claude Code 설치

AI 기반 코딩 어시스턴트 Claude Code를 설치하여 개발 생산성을 극대화합니다.

npm install -g @anthropic-ai/claude-code
claude

Claude 소개

Anthropic이 개발한 안전하고 강력한 AI 어시스턴트

Claude란?

Claude는 AI 안전 회사 Anthropic이 개발한 대형 언어 모델(LLM) 기반의 AI 어시스턴트입니다. 2023년 출시 이후 지속적인 업데이트를 통해 코딩, 분석, 창작, 대화 등 다양한 분야에서 탁월한 성능을 보여주고 있습니다.

안전성 우선

Constitutional AI 기법으로 안전하고 윤리적인 응답을 생성합니다.

코딩 능력

Python, JavaScript, TypeScript 등 다양한 언어의 코드를 작성하고 디버깅합니다.

긴 컨텍스트

최대 200K 토큰의 컨텍스트 윈도우로 긴 문서를 처리합니다.

다국어 지원

한국어를 포함한 다양한 언어로 자연스러운 대화가 가능합니다.

Claude 모델 라인업

모델특징적합한 용도
Claude Opus 4가장 강력한 모델복잡한 분석, 연구, 고급 코딩
Claude Sonnet 4성능과 속도의 균형일반적인 코딩, 글쓰기, 분석
Claude Haiku 4빠르고 경량간단한 작업, 빠른 응답 필요 시

Claude 설치방법

Claude를 사용하는 다양한 방법

방법 1: Claude.ai 웹 사용

가장 간단한 방법입니다. 브라우저에서 바로 사용할 수 있습니다.

주소: https://claude.ai — 회원가입 후 무료로 사용 가능 (유료 플랜으로 더 많은 기능 이용)

방법 2: Claude Code CLI 설치

터미널에서 Claude를 직접 사용하는 강력한 도구입니다.

npm install -g @anthropic-ai/claude-code
claude --version

팁: Claude Code를 프로젝트 디렉토리에서 실행하면 코드베이스를 이해하고 더 정확한 도움을 받을 수 있습니다.

방법 3: API 키 설정

개발자 도구에서 Claude API를 사용하려면 API 키가 필요합니다.

export ANTHROPIC_API_KEY="your-api-key-here"

기본사용법

Claude를 효과적으로 사용하는 기본 방법

대화 시작하기

Claude와의 대화는 자연스러운 언어로 질문하면 됩니다. 명확하고 구체적으로 요청할수록 더 좋은 결과를 얻을 수 있습니다.

좋은 질문 예시: "Python으로 CSV 파일을 읽어서 각 행의 데이터를 JSON 형식으로 변환하는 함수를 작성해줘. 에러 처리도 포함해줘."

고급기능

Claude의 강력한 고급 기능들을 활용하세요

Projects 기능

Claude.ai의 Projects 기능을 사용하면 대화 내용을 프로젝트별로 관리하고, 프로젝트 컨텍스트를 지속적으로 유지할 수 있습니다.

  • 프로젝트별 커스텀 지시사항 설정
  • 파일 및 문서 업로드로 컨텍스트 확장
  • 팀원과 프로젝트 공유

Artifacts

Claude는 코드, HTML, 마크다운 등 다양한 형식의 결과물을 Artifact로 생성할 수 있습니다. Artifact는 독립적인 창에 표시되어 바로 미리보기가 가능합니다.

Vision (이미지 이해)

Claude는 이미지를 분석하고 설명할 수 있습니다. 스크린샷, 다이어그램, UI 디자인 등을 업로드하면 내용을 이해하고 관련 코드나 설명을 제공합니다.

Tool Use (함수 호출)

API를 통해 Claude에게 외부 도구나 함수를 사용할 수 있는 능력을 부여할 수 있습니다. 이를 통해 데이터베이스 조회, 웹 검색, 계산 등의 작업을 자동화할 수 있습니다.

프롬프트 엔지니어링

Claude에게 최적의 결과를 이끌어내는 프롬프트 작성법

기본 원칙

구체적으로 요청하기

모호한 요청보다는 명확하고 구체적인 요청이 더 좋은 결과를 만듭니다.

컨텍스트 제공

작업의 배경, 목적, 제약 조건을 미리 설명하면 더 적합한 답변을 받을 수 있습니다.

역할 설정

"당신은 시니어 백엔드 개발자입니다"와 같이 역할을 부여하면 전문적인 답변을 받습니다.

출력 형식 지정

마크다운, JSON, 코드 블록 등 원하는 출력 형식을 명시하세요.

효과적인 프롬프트 예시

# 좋은 프롬프트 예시

역할: 당신은 10년 경력의 Node.js 백엔드 개발자입니다.

작업: Express.js로 사용자 인증 미들웨어를 작성해주세요.

요구사항:
- JWT 토큰 검증
- 에러 처리 포함
- TypeScript 사용
- 코드에 주석 추가

출력 형식: 코드 블록으로 제공해주세요.

Chain of Thought (단계별 사고)

복잡한 문제는 "단계적으로 생각해서 설명해줘"라고 요청하면 더 정확한 결과를 얻을 수 있습니다. Claude는 각 단계를 명시하면서 더 논리적인 추론을 합니다.

토큰 절감방안

효율적인 Claude 사용으로 토큰을 절약하는 방법

토큰이란?

토큰은 Claude가 텍스트를 처리하는 단위입니다. 대략 1토큰 ≈ 0.75 영어 단어 (한국어는 더 많은 토큰 소비). 토큰 사용량은 API 비용에 직결됩니다.

토큰 절감 전략

🗜️ /compact 사용

Claude Code에서 /compact 명령으로 대화 기록을 요약하여 컨텍스트를 줄입니다.

📋 CLAUDE.md 활용

프로젝트 컨텍스트를 CLAUDE.md에 기록하여 매번 설명하는 수고를 줄입니다.

🎯 핵심만 요청

불필요한 설명 없이 핵심 내용만 요청하면 입출력 토큰 모두 절감됩니다.

🔄 캐싱 활용

Prompt Caching 기능으로 반복되는 시스템 프롬프트의 토큰 비용을 90% 절감합니다.

Prompt Caching

자주 사용하는 긴 시스템 프롬프트나 문서가 있다면 Prompt Caching을 활용하세요. 처음에만 처리 비용이 발생하고 이후에는 95% 저렴한 캐시 비용만 지불합니다.

{
  "system": [
    {
      "type": "text",
      "text": "긴 시스템 프롬프트...",
      "cache_control": {"type": "ephemeral"}
    }
  ]
}

Anthropic 이야기

Claude를 만든 AI 안전 회사 Anthropic에 대해

Anthropic 소개

Anthropic은 2021년 Dario Amodei와 Daniela Amodei 남매를 중심으로 OpenAI 출신 연구자들이 설립한 AI 안전 회사입니다. "안전하고 유익한 AI 시스템 구축"을 미션으로 합니다.

핵심 기술: Constitutional AI

Anthropic의 가장 중요한 연구 성과 중 하나는 Constitutional AI(CAI)입니다. 이는 AI가 인간이 설정한 원칙(constitution)에 따라 스스로 행동을 평가하고 개선하는 방식입니다.

GitHub 소개

세계 최대의 코드 호스팅 및 협업 플랫폼

GitHub란?

GitHub는 Git 버전 관리 시스템을 기반으로 하는 코드 호스팅 플랫폼입니다. 2008년 설립되어 2018년 Microsoft에 인수되었으며, 현재 1억 명 이상의 개발자와 4억 개 이상의 저장소를 보유하고 있습니다.

버전 관리

Git을 통한 코드 변경 이력 관리 및 브랜치 전략을 지원합니다.

협업

Pull Request, Code Review, Issues로 팀 협업을 효율화합니다.

CI/CD

GitHub Actions로 자동 빌드, 테스트, 배포 파이프라인을 구축합니다.

보안

Dependabot, Secret Scanning, Code Scanning으로 보안을 강화합니다.

협업 워크플로우

팀과 함께 효율적으로 개발하는 방법

Fork & Pull Request 워크플로우

  1. 원본 저장소를 Fork합니다
  2. 포크한 저장소를 로컬에 Clone합니다
  3. 새 브랜치를 만들고 변경사항을 커밋합니다
  4. 포크한 저장소에 Push합니다
  5. 원본 저장소에 Pull Request를 생성합니다
  6. 코드 리뷰 후 Merge합니다
git checkout -b feature/new-feature
git push origin feature/new-feature
gh pr create --title "Add new feature" --body "Description"

코드 리뷰 모범 사례

  • PR은 작고 집중적으로 유지하세요 (300줄 이하 권장)
  • 명확한 PR 설명과 스크린샷을 첨부하세요
  • 리뷰는 코드에 대한 것이지 사람에 대한 것이 아닙니다
  • LGTM(Looks Good To Me) 전 최소 1명 이상 리뷰를 받으세요

GitHub Flow

간결하고 효과적인 GitHub 브랜치 전략

GitHub Flow란?

GitHub Flow는 main 브랜치 하나를 기반으로 하는 간단한 브랜치 전략입니다. 지속적 배포(CD)에 최적화되어 있습니다.

1. 브랜치 생성

main에서 기능 브랜치를 만듭니다. 이름은 descriptive하게 작성합니다.

2. 커밋

작은 단위로 자주 커밋합니다. 커밋 메시지는 명확하게 작성합니다.

3. Pull Request

리뷰가 필요할 때 PR을 오픈합니다. 완성 전에도 Draft PR을 활용합니다.

4. 리뷰 & 머지

팀원의 리뷰 후 main에 머지합니다. 자동 배포가 트리거됩니다.

브랜치 네이밍 컨벤션

feature/새 기능 개발feature/user-auth
fix/버그 수정fix/login-error
docs/문서 수정docs/api-guide
refactor/코드 리팩토링refactor/db-layer
hotfix/긴급 수정hotfix/security-patch

GitHub Actions

자동화된 CI/CD 파이프라인 구축

GitHub Actions란?

코드 변경 시 자동으로 빌드, 테스트, 배포하는 CI/CD 플랫폼입니다. 저장소의 .github/workflows/ 디렉토리에 YAML 파일로 워크플로우를 정의합니다.

기본 워크플로우 예시

name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm test
      - run: npm run build

Supabase란?

Supabase는 PostgreSQL을 기반으로 한 오픈소스 백엔드 서비스(BaaS)입니다. 데이터베이스, 인증, 스토리지, 실시간 기능, 엣지 함수를 하나의 플랫폼에서 제공합니다.

PostgreSQL

실시간 기능

자동 API

데이터베이스 스키마에서 RESTful API와 GraphQL을 자동으로 생성합니다.

오픈소스

RLS & 보안

Row Level Security로 데이터 접근 제어하기

RLS란?

Row Level Security(RLS)는 PostgreSQL의 기능으로, 각 행(row)에 대한 접근을 사용자별로 제어할 수 있습니다. Supabase에서는 RLS를 통해 사용자가 자신의 데이터에만 접근하도록 제한합니다.

중요: Supabase에서는 새 테이블을 만들 때 반드시 RLS를 활성화하세요. 기본적으로 비활성화 상태라면 모든 anon 사용자가 데이터에 접근할 수 있습니다.

RLS 정책 예시

-- RLS 활성화
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

-- 사용자는 자신의 글만 볼 수 있음
CREATE POLICY "Users can view own posts"
ON posts FOR SELECT
USING (auth.uid() = user_id);

-- 인증된 사용자만 글 작성 가능
CREATE POLICY "Authenticated users can insert"
ON posts FOR INSERT
WITH CHECK (auth.uid() = user_id);

-- 자신의 글만 수정/삭제 가능
CREATE POLICY "Users can update own posts"
ON posts FOR UPDATE
USING (auth.uid() = user_id);

벡터DB & AI

pgvector로 AI 기반 시맨틱 검색 구현하기

pgvector란?

Supabase는 PostgreSQL 확장인 pgvector를 지원하여 벡터 데이터를 저장하고 유사도 검색을 수행할 수 있습니다. RAG(Retrieval Augmented Generation) 시스템 구축에 활용됩니다.

벡터 테이블 생성 및 검색

-- pgvector 확장 활성화
CREATE EXTENSION vector;

-- 벡터 컬럼이 있는 테이블
CREATE TABLE documents (
  id BIGSERIAL PRIMARY KEY,
  content TEXT,
  embedding VECTOR(1536)
);

-- 유사도 검색 함수
CREATE OR REPLACE FUNCTION match_documents(
  query_embedding VECTOR(1536),
  match_threshold FLOAT,
  match_count INT
)
RETURNS TABLE(id BIGINT, content TEXT, similarity FLOAT)
LANGUAGE SQL AS $$
  SELECT id, content,
    1 - (embedding <=> query_embedding) AS similarity
  FROM documents
  WHERE 1 - (embedding <=> query_embedding) > match_threshold
  ORDER BY similarity DESC
  LIMIT match_count;
$$;

Vercel 소개

프론트엔드 배포의 표준 플랫폼

Vercel이란?

Vercel은 프론트엔드 웹 애플리케이션을 위한 클라우드 플랫폼입니다. Next.js를 만든 회사로, GitHub와 연동하여 코드 변경 시 자동으로 배포하는 CI/CD 기능을 제공합니다.

자동 배포

GitHub push만으로 자동 배포. PR마다 미리보기 URL도 자동 생성됩니다.

엣지 네트워크

전 세계 엣지 서버에 배포하여 어디서나 빠른 로딩 속도를 제공합니다.

Serverless Functions

API 라우트를 서버리스 함수로 자동 변환하여 별도 백엔드 없이 API 구현이 가능합니다.

Analytics

실시간 웹 성능 분석 및 Core Web Vitals 모니터링을 기본 제공합니다.

자주 묻는 질문

자주 나오는 질문들에 대한 답변

Claude는 무료로 사용할 수 있나요?
Claude.ai에서 무료 플랜으로 사용할 수 있습니다. 무료 플랜은 사용량에 제한이 있으며, Pro 플랜($20/월)으로 업그레이드하면 더 많은 메시지와 고급 모델을 사용할 수 있습니다.
Supabase 무료 플랜의 한계는?
무료 플랜은 500MB 데이터베이스, 1GB 스토리지, 50,000 MAU(월 활성 사용자)를 제공합니다. 소규모 프로젝트나 테스트에 충분하며, 프로덕션 수준의 서비스는 Pro 플랜($25/월)을 권장합니다.
Vercel 무료 플랜으로 상업적 사용이 가능한가요?
Vercel Hobby 플랜(무료)은 개인 비상업적 프로젝트에 사용하도록 설계되어 있습니다. 상업적 프로젝트는 Pro 플랜($20/월)을 사용해야 합니다.
GitHub Actions는 무료인가요?
공개 저장소는 GitHub Actions가 완전 무료입니다. 비공개 저장소는 매월 2,000분의 무료 사용량을 제공하며, 초과 시 요금이 발생합니다.
VS Code와 GitHub Copilot의 차이는?
VS Code는 코드 에디터이고, GitHub Copilot은 VS Code에 설치하는 AI 코드 자동완성 확장 프로그램입니다. Copilot은 $10/월의 구독 비용이 있지만, 학생과 오픈소스 기여자는 무료로 사용할 수 있습니다.
